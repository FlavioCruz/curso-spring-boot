# Curso Introdução Spring-Boot

O presente código e documentação serve de material para o curso de introdução ao spring-boot.
O objetivo do curso é ensinar interessados no desenvolvimento Web usando java, spring-boot e testes.

Os tópicos que são abordados são:

* Java 8;
* Padrão MVC;
* Testes;
    * Junit 5;
    * Jupiter-Webdriver;
    * Teste nos spring-boot;
* Spring-Boot
    * IOC e DI;
    * Anotações (@Service, @Autowired, @Repository, ...)
    * REST API;
    * Controller;
    * Thymeleaf;

    
É objetivo ser um curso introdutório e não um curso completo. Será apresentada uma referência bibliográfica 
com outros materiais para aprofundar o tema;

## Pré requisitos

Aconselha-se a conhecer previamente os tópicos:

* Programação estruturada, procedural e orientada a objetos;
* SQL e banco de dados;
* Java básico;

O CEDERJ disponibiliza [vídeos](http://www.cederj.edu.br/videoaulas) de cursos completos de graduação para os tópicos 
teóricos.
 
Para java a Caelum oferece [material introdutório](https://www.caelum.com.br/apostila-java-orientacao-objetos) de java

## Tópicos importantes não abordados

Nós não abordamos propositalmente de [spring-data](https://docs.spring.io/spring-data/jpa/docs/current/reference/html) 
e nem jpa-hibernate. Pois embora tais ferramentas aumentem a velocidade de desenvolvimento, acreditamos que
é preciso de um curso próprio para estes temas sejam abordados de forma satisfatória. Além disto, JPA não é uma solução 
trivial e exige maturidade do usuário em programação com banco de dados.

Também não abordamos Javascript e CSS, pois este curso não aborda a fundo a camada de View de uma aplicação Web. 
É preciso um curso dedicado aos temas.

Embora o curso utilize testes para o treinamento, ele não foca na explicação de teoria nem na prática de testes ou TDD.
Isto foi feito propositalmente para não extrapolar o tempo desejado de curso. Porém encorajamos as pessoas após usarem 
o curso investiguem teoria de teste. Materiais importantes são:

* [TDD by Example 2002, Berk](https://www.amazon.com.br/Test-Driven-Development-Kent-Beck/dp/0321146530)
* [Spring-Boot Test Reference](https://docs.spring.io/spring-boot/docs/current/reference/html/boot-features-testing.html)   
* [Spring Test Reference](https://docs.spring.io/spring/docs/current/spring-framework-reference/testing.html)
* [XUnit Patterns](http://xunitpatterns.com)

## Metodologia

Para uma aula presencial com tutor/professor, recomendamos 3 horas. A metodologia recomendada é a apresentação dos problemas,
apresentação teórica dos conceitos a serem usados e a solução dos testes.

Para aprendizagem individual, recomendamos a ler o material didático e os materiais de apoio, acreditamos que leve em média
12 horas.

Acreditamos que uma versão deste curso pode ser realizada em grupo, através de um rodízio de quem realiza 
a solução dos problemas, sobre a orienção de um tutor. E o estudo dos materiais didáticos base.
Isto deve levar 9 horas.
